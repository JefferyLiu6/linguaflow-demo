# LinguaFlow Agent v1 — Implementation Reference

Snapshot of the Python agent before the v2 upgrade.

---

## Overview

Standalone FastAPI service that handles all LLM logic for LinguaFlow. The Next.js app never calls an LLM directly — it proxies requests here after handling auth and rate limiting.

```
Next.js API route → POST /generate | POST /tutor | POST /tutor/stream
```

Run:
```bash
uvicorn main:app --host 0.0.0.0 --port 8000 --reload
```

---

## File Layout

```
agent/
  main.py          FastAPI app, CORS, router mounts, /health
  providers.py     LLM factory — resolves "provider/model-name" → LangChain model
  generation.py    /generate endpoint — single-pass LLM + JSON extraction
  config.py        Shared env vars (OLLAMA_BASE_URL, DEFAULT_MODEL)
  requirements.txt
  .env.example     Agent-specific keys (never in Next.js .env)
  tutor/
    schemas.py     Pydantic request/response models + LangGraph TypedDict state
    nodes.py       Router node + 5 specialist nodes + feedback/streaming helpers
    graph.py       LangGraph StateGraph assembly + /tutor + /tutor/stream endpoints
```

## Endpoints

### `GET /health`
Returns agent status, configured providers, default model, and Ollama base URL.

### `POST /generate`
Single-pass drill generation. No LangGraph — just one `ainvoke` call.

**Request** (`GenerateRequest`):
```json
{
  "mode": "guided | raw",
  "language": "Spanish",
  "count": 10,
  "model": "openai/gpt-4o-mini",
  "guided": {
    "topic": "daily | tech | finance | business",
    "difficulty": "a1 | b1 | c1 | c2",
    "grammar": "mixed | subjunctive | conditional | pastperf",
    "drill_type": "translation | substitution | transformation"
  },
  "raw_prompt": ""
}
```

**Response** (`GenerateResponse`):
```json
{
  "drills": [{ "id", "type", "instruction", "prompt", "answer", "prompt_lang" }],
  "model": "openai/gpt-4o-mini",
  "elapsed_ms": 1240
}
```

**Flow:**
1. Build user prompt from `guided` params or pass through `raw_prompt`
2. Call `get_llm(model).ainvoke([SystemMessage, HumanMessage])`
3. Strip markdown fences, extract first JSON array (`extract_json`)
4. Validate each item has `prompt` + `answer`; coerce unknown `type` to `"translation"`

System prompt instructs the model to output a raw JSON array only — no fences, no preamble. `extract_json` handles models that add them anyway.

### `POST /tutor`
LangGraph-based coaching. Stateless per request — full conversation history sent each time.

**Request** (`TutorRequest`):
```json
{
  "mode": "feedback | tutor",
  "model": "openai/gpt-4o-mini",
  "session_context": { "language", "drill_type", "item_index", "items_total" },
  "current_item": { "instruction", "prompt", "type", "expected_answer", "user_answer", "feedback" },
  "recent_items": [{ "prompt", "user_answer", "correct", "timed_out" }],
  "messages": [{ "role": "user | assistant", "content": "..." }],
  "constraints": { "max_coach_turns": 10, "max_hint_level": 3, "current_hint_level": 0 }
}
```

**Response** (`TutorResponse`):
```json
{
  "assistant_message": "...",
  "structured": { "hint_level", "suggested_phrase", "learner_ready" },
  "model": "openai/gpt-4o-mini",
  "elapsed_ms": 980
}
```

**Turn cap:** enforced before LLM invocation — returns a canned "limit reached" message at `max_coach_turns` user turns.

---

### `POST /tutor/stream`
Same as `/tutor` but returns `text/event-stream` (SSE).

Each chunk: `data: {"token": "..."}\n\n`
Final chunk: `data: {"done": true, "route": "hint", "hint_level": 2}\n\n`
Errors: `data: {"error": "..."}\n\n`

`mode: "feedback"` streams the feedback node directly (no router).  
`mode: "tutor"` runs the router synchronously first, then streams the specialist.

## LangGraph Tutor Graph

```
[router_node]
     │
     └─ conditional edges ─┬─ hint_node
                           ├─ socratic_node
                           ├─ explain_node
                           ├─ clarify_node
                           └─ ready_check_node
                                     │
                                    END
```

### Router node
Calls the LLM with `_ROUTER_SYSTEM` prompt — outputs `{"route": "<route>"}` JSON only.
Falls back to `"socratic"` on parse failure.

### Specialist nodes
All five call a shared `_run_specialist(state, route)` helper which:
1. Builds a system prompt specific to that coaching style
2. Calls `get_llm(model).ainvoke(messages)`
3. Writes `assistant_message` (and optional `structured_data`) back to state

| Route | Coaching style |
|---|---|
| `hint` | Small nudge toward the answer; increments `hint_level` |
| `socratic` | Guiding question (default) |
| `explain` | Grammar rule or pattern explanation |
| `clarify` | Explain what the drill is asking |
| `ready_check` | Confirm learner is ready to move on |

### Feedback mode
Separate prompt (`_FEEDBACK_PROMPT`) — identifies the error category and explains the violated rule. Never reveals the expected answer. Used immediately after a drill answer before the coaching conversation begins.

## Provider Factory (`providers.py`)

Format: `"provider/model-name"` (bare names treated as Ollama, for backward compat).

| Provider prefix | LangChain class | Key env var |
|---|---|---|
| `openai` | `ChatOpenAI` | `OPENAI_API_KEY` |
| `anthropic` | `ChatAnthropic` | `ANTHROPIC_API_KEY` |
| `google` | `ChatGoogleGenerativeAI` | `GOOGLE_API_KEY` |
| `groq` | `ChatGroq` | `GROQ_API_KEY` |
| `ollama` | `ChatOllama` | none (`OLLAMA_BASE_URL`, default `http://localhost:11434`) |
| `ollama-cloud` | `ChatOllama` | `OLLAMA_API_KEY` (base: `https://ollama.com`, Bearer auth via `client_kwargs`) |

All providers are installed at startup (`requirements.txt`); each is only imported when its branch is reached. Unknown providers raise `ValueError` immediately.

## Key Design Decisions

**Single-pass generation, no LangGraph** — `/generate` is a simple request–response chain. LangGraph overhead is not justified for a one-shot JSON extraction task.

**Stateless tutor** — full conversation history is sent on every request. Simplifies deployment (no session store) at the cost of token usage on long conversations. Turn cap (`max_coach_turns`) limits runaway cost.

**Never reveal the answer** — the feedback prompt is written to identify error *categories* and *rules* rather than stating the correct form. Specialist prompts follow the same constraint.

**JSON-only generation prompt** — the system prompt demands a raw JSON array with no markdown. `extract_json` strips fences and finds the first `[…]` substring as a fallback for models that disobey.

**Ollama cloud via `client_kwargs`** — `ChatOllama` accepts `client_kwargs` which are forwarded to the underlying `ollama.AsyncClient`. This lets us inject the `Authorization` header without subclassing or patching.

**All providers installed, lazily imported** — `requirements.txt` lists all LangChain provider packages. Each is imported inside its `if provider == "..."` branch, so missing keys produce a `ValueError` at call time rather than an import error at startup.
